AlertQuery

public struct AlertQuery

Control how to search alerts. By default, it uses the current location and beam mode with default values. When passing route the search will be done in coordinates instead. By default, the current location altitude will be used to make a range with +/- 2k feet. Set altRange when you need the custom range to search for alerts.

  • usePolygonsEnabled

    By default it’s false and only route corridor and viewport low-resolution H3 turbulence reports will be queried. But when offline and no low-resolution turbulence reports are cached it could be useful to use high-resolution turbulence polygons to search for alerts instead. In this case, set it to true and the query will return a TurbulenceItem array generated from turbulence polygons cached for the specified area. It will use polygons data only if no low-resolution H3 turbulence reports meet the query. A recommendation for when to use it is when the app is offline. Optional. By default, it is false.
